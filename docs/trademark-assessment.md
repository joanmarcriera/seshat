# Trademark / Name-Availability Notes — "Seshat" (macOS app)

**Subject:** A macOS menu-bar app named **Seshat** that turns meeting recordings into Markdown
notes via the user's own WhisperX + Ollama servers (local-first, bring-your-own-server).

**Status:** Informal, evidence-based screening — **not legal advice** and **not a formal clearance
search.** Complete the checklist in §3 before any store submission.

---

## 1. Why the name is "Seshat"

An earlier working name was dropped before launch because it sat inside the crowded, easily-confused
"Scribe / transcription" cluster (a directly-competing commercial product was already trading under
a near-identical name, plus a near-homophone of a well-known registered mark). The product was
renamed to **Seshat** — the ancient Egyptian goddess of writing and record-keeping — which is
distinctive in this niche and avoids that cluster entirely.

## 2. Initial screening on "Seshat"

- No macOS transcription / meeting-notes app was found using the name "Seshat" in initial App Store
  and web screening.
- "Seshat" is a mythological / common-cultural term, which generally weakens any single party's
  claim to it as a software mark — but it is also used by unrelated projects/products in other
  fields, so a **formal clearance in the relevant classes is still required** (this is the
  outstanding hard gate before submission).
- Bundle IDs are already `uk.co.riera.seshat` / `uk.co.riera.seshat-setapp`; the GitHub repo is
  `joanmarcriera/seshat`.

**Verdict:** likely lower risk than the prior name, but **unconfirmed** — do the formal search below.

## 3. Before-submission checklist

- [ ] **Formal clearance search** on "Seshat" — query the live registers directly (not mirrors):
      USPTO Trademark Search (tmsearch.uspto.gov), UK IPO (gov.uk/search-for-trademark), EUIPO
      eSearch plus, and **TMview** (tmview.org) across the EU/UK/US in **Class 9** (downloadable
      software) and **Class 42** (SaaS, if any hosted component). Consider a paid clearance/watch
      service or a trademark attorney for a few hundred GBP — cheap insurance vs. a post-launch
      rebrand.
- [ ] **App Store name check** — search App Store Connect / the live App Store for "Seshat" and
      close variants; confirm no confusingly-similar app and that the name is available for your
      bundle ID and listing.
- [ ] **Domain + handle sweep** — secure `.app` and/or `.com` plus a fallback, and matching GitHub
      org and social handles, before announcing.
- [ ] **Common-law / web search** — Google "Seshat" + "app", "transcription", "notes" to catch
      unregistered but active users (common-law rights) that a register search misses.
- [ ] **Decide on filing** — once cleared, consider a **UK IPO** filing (cheapest home register for
      a UK-based developer) and optionally **USPTO** (intent-to-use) and an **EUTM** if selling into
      the EU. File in **Class 9** and **Class 42**. File only after clearance confirms the name is
      open.
- [ ] **Document the clearance** (screenshots/dates) so you have evidence of good-faith adoption.
